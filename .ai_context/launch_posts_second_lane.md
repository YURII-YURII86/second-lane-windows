# Second Lane Launch Posts

## GitHub / General Launch Post

### EN Draft

Claude Code hit a limit before the task was done.

I did not want to wait, lose context, or restart from scratch.

So I moved the same task into ChatGPT, connected it to my machine through GPT Actions, and kept going: inspect the project, patch code, run commands, verify the result.

That became Second Lane.

Second Lane is an open-source way to keep building in ChatGPT when Claude Code, Codex, or another AI coding agent hits its limits.

### RU Draft

Claude Code упёрся в лимит до завершения задачи.

Так появился Second Lane.

Second Lane — это open-source способ продолжать разработку в ChatGPT, когда Claude Code, Codex или другой AI coding-agent больше не может продолжать.

## Reddit Post Draft

### EN Draft

I built a tool for a very specific problem: Claude Code / Codex is great until it hits a limit, and then your momentum dies.

Second Lane lets me keep going inside ChatGPT, but with real actions on my machine.

The point is not to replace IDE agents.
The point is to have a second execution lane when they stop.

### RU Draft

Сделал инструмент под очень конкретную боль: Claude Code / Codex хорош ровно до тех пор, пока не упрётся в лимит, а потом умирает темп работы.

Second Lane позволяет продолжить внутри ChatGPT, но уже с реальными действиями на моей машине.

Смысл не в том, чтобы заменить IDE-агентов.
Смысл в том, чтобы иметь второй рабочий контур, когда они останавливаются.
